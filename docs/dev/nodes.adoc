== Perception

[[node-depth-camera-node]]
=== depth_camera_node (Python/C++)

*Package*:: `perception_pkg` (Python), `perception_camera_cpp` (C++)
*Executable*:: `depth_camera_node`
*Language*:: Python / C++
*Launch*:: `bringup/launch/perception.launch.py` (`depth_camera_front`, `depth_camera_rear`)
*Subscriptions*:: (none)
*Publications*:: <<topics.adoc#topic-dori-camera-front-color-image-raw,/camera/front/color/image_raw>> (`sensor_msgs/msg/Image`), <<topics.adoc#topic-dori-camera-front-depth-image-raw,/camera/front/depth/image_raw>> (`sensor_msgs/msg/Image`), <<topics.adoc#topic-dori-camera-front-depth-image_colormap,/camera/front/depth/image_colormap>> (`sensor_msgs/msg/Image`, Python impl), <<topics.adoc#topic-dori-camera-front-color-camera_info,/camera/front/color/camera_info>> (`sensor_msgs/msg/CameraInfo`), <<topics.adoc#topic-dori-camera-front-depth-camera_info,/camera/front/depth/camera_info>> (`sensor_msgs/msg/CameraInfo`), <<topics.adoc#topic-dori-camera-front-depth-scale,/camera/front/depth/scale>> (`std_msgs/msg/Float32`)
*Services (server)*:: -
*Services (client)*:: -
*Actions*:: -
*Parameters*:: See <<parameters.adoc#param-depth_camera_node, depth_camera_node parameters>>
*Lifecycle*:: unmanaged
*Dependencies*:: Intel RealSense device, `pyrealsense2` (Python) or librealsense (C++)
*Failure behavior*:: If device open/stream start fails, logs an error and the node exits.

*Description*:: Source node for front/rear camera streams. In bringup, the C++ implementation is selected by default and remapped to `/camera/front/...` and `/camera/rear/...`.
