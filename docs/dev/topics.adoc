= ROS2 Topic API Reference

This is a single source (SSOT) for ROS2 topic details documentation.

* Topic anchors use the `topic-...` format, and `/` in topic paths is replaced with `-`.
* Topic details are maintained in the common format below.
* The details list is not duplicated in `architecture.md`; instead, this document is referenced.

== Template

[[topic-template-example]]
/topic/name::
+
--
*Type (msg)*:: `pkg/msg/Type`

*Publishers / Subscribers*::
+
[source,text]
----
Publishers
- publisher_node_a

Subscribers
- subscriber_node_a
- subscriber_node_b
----

*Frequency*:: `N/A (measure and update)`

.Message fields
[cols="1,1,1,3",options="header"]
|===
| Field | Type | Unit | Description
| data | string | - | Meaning of the message payload.
|===

*Description*:: Topic purpose, lifecycle, and caveats (QoS/latency/trigger conditions).
--

== Perception

[[topic-dori-camera-color-image-raw]]
/dori/camera/color/image_raw::
+
--
*Type (msg)*:: `sensor_msgs/msg/Image`

*Publishers / Subscribers*::
+
[source,text]
----
Publishers
- depth_camera_node

Subscribers
- person_detection_node
- gesture_recognition_node
- facial_expression_node
- landmark_detection_node
----

*Frequency*:: `TBD (measure and update)`

.Message fields
[cols="1,1,1,3",options="header"]
|===
| Field | Type | Unit | Description
| header | std_msgs/msg/Header | - | Frame timestamp and frame_id.
| height | uint32 | px | Image height.
| width | uint32 | px | Image width.
| encoding | string | - | Pixel encoding format.
| is_bigendian | uint8 | - | Endianness flag.
| step | uint32 | bytes | Full row length in bytes.
| data | uint8[] | bytes | Raw image buffer.
|===

*Description*:: RGB camera stream used by perception pipelines.
